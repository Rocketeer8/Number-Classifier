Introduction

In A1 we saw that kNN was very accurate, but also very slow when performing handwriting recognition. In this assignment we will try to speed up kNN by writing a parallel implementation to make use of the fact that most modern computers have multiple processors over which we can spread the work. 

Just like in A1 the goal is to classify some large number of images that make up the test dataset and compute kNN's accuracy over this dataset. Instead of having one process that sequentially produces the labels for all test images and then computes the accuracy, a parent process will divide up the work among several child processes. The parent will use pipes to tell each child which images in the test dataset it is in charge of. The child will use kNN to classify the images that it was assigned by the parent and then uses a pipe to report back how many of those images it classified correctly and how many it classified incorrectly. The parent will compute and output the accuracy across the entire test data set based on the reports it receives from its children.

 

Program Details

The program you will write takes 4 command line arguments:

- The value k of the kNN algorithm

- The name of the file containing the test set of images

- The name of the file containing the training set of images

- The number of children that the parent process will create to classify the images in the training set.

The only output that the program will produce is the accuracy achieved by kNN when classifying the images in the test data set.

 

The Format of the Input Data

The two files that comprise the training and the test data sets are in the same binary format as in A2. The only difference in the binary files is that this time the pixels have not all been converted to 0/255, and instead have all the possible colors.

Analyzing the speedup from parallelization

An interesting parameter when running your program is the number of children the parent uses. In an ideal world, using two parallel processes instead of one sequential process will allow us to classify the same number of images in half the time. Or more generally, using n  times the number of processing lets us finish classification n  times faster. Run your program for n = 2, 3, 4, 5, .. 10 child processes and report the time classification took for each value of n. You can use the time command in bash to measure the running time of your program. Do you observe a linear speed-up, where k  child processes finish the job k  times faster? If not, why do you think that is the case? Is there a limit to how big k  can be before this stops happening, and if so why?

 

Important Hints 

Here are a few things to remember as you implement and test the functions:

  1. Test thoroughly before submitting a final version. We are using automated grading tools, so your program must compile and run according to the specifications. In particular, your program must produce output in the specified format.
  2. As always, build up functionality one small piece at a time.
  3. Check for errors on every system or library call that is not a print. Send any potential error messages to stderr.
  4. Be very careful to initialize pointers to NULL and to make sure that you copy any strings that you use.
  5. You will be reusing significant amounts of code from A1 and A2 for this assignment, however we will not be testing to see if the kNN / dataset specific code is correct again. You just need to make sure that or instance calling the loaddataset() function or kpredict() function will not crash the code.
